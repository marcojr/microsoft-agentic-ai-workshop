Important architecture decision
-------------------------------

Semantic Kernel is no longer part of the active runtime.

The Draft Agent was migrated to Microsoft Agent Framework.

All active pro-code LLM agents now use Microsoft Agent Framework or deterministic MCP-backed execution.

Current runtime flow:

```text
Copilot Studio / Power Apps / API caller
  -> Azure Function Orchestrator API
  -> Microsoft Agent Framework agents
  -> MCP client/tools
  -> Dataverse / Azure AI Search / Service Bus / App Insights
```

Current next phases
-------------------

Roadmap file:

```text
docs/next-phases.md
```

Phases:

1. Copilot Studio Modernization
2. Microsoft 365 Agents SDK
3. RAG / Knowledge Expansion
4. Power Platform / Tool Ecosystem Expansion
5. Microsoft Purview Governance

Phase 2 is important.

The current Copilot Studio agent is still mostly a front door. The next phase should make modern Copilot Studio own more conversation orchestration:

- tools
- skills
- knowledge
- connected agents
- memory/state where available
- routing between approval review, policy lookup and full backend workflow

But the backend still owns:

- Dataverse writes
- approval state transitions
- cost/telemetry logging
- governance decisions
- official workflow summaries
- reusable workflow execution for future Microsoft 365 Agents SDK

Planned Copilot Studio skills:

- `Review Pending Approvals`
- `Submit Approval Decision`
- `Explain Approval Risk`
- `Order Support Triage`
- `Policy Lookup`
- `Escalation Recommendation`
